What is the Ecoindex?

Ecoindex.fr is an online service designed and offered by Frédéric Bordage (Greenit.fr) and directed by Gwenael Chailleu and Thomas Samain of the company NXTWEB.

This service aims to identify the environmental impact of a website. It provides the user several indicators:

  • The famous Ecoindex: environmental performance (100 score, Higher is Better)
  • A note from A to G reflecting this performance
  • Its technical imprint (weight, complexity)
  • Its environmental footprint (greenhouse gas, water)
  • A list of good practice as well as an assessment of respect for it

The objective of the service is an awareness of the environmental impact of digital services, as well as the concrete tracking proposal.

Technical imprint

The technical footprint of a site is represented by several indicators:

  • The size of the DOM: This indicator makes it possible to assess the complexity of a website. The greater the DOM, the more it will need power and memory for its load as well as for all operations concerning it.
  • The loaded file weight: This indicator assesses the impact of the loading of the website on the network. The more heavy the files will be the more they will require network resources.
  • The number of requests made to loading the page: this indicator allows you to assess how much the loading of the page requires servers. This number allows the consumption of servers to be interpolated.

Environmental imprint

The environmental footprint is broken down into two points

  • Greenhouse gas emission, measured in equivalent CO2 gram
  • Water consumption, measured in CL

These values come from an estimate taking into account all the actors (customer, network and server), and are based on an analysis of the life cycle of the different devices.

Good practices

The CNUMR offers a good practice list for web platform design. Certain points on this list can be tested automatically during the platform analysis.

A non-exhaustive list of good practices tested:

  • The number of JavaScript plugin used.
  • The number of CSS files. It can be useful to cut your CSS into several specialized files. This cut can be made according to a functional logic (general content, page A, page B, etc.) or a query media logic (a "print" page for example). The Ecoindex an end in it?

Is a good eco-is eco-responsible page? The Ecoindex is a good indicator, but it is often necessary to go further in an eco -design process. For example, the Ecoindex does not allow you to test the functional sobriety of a website. Indeed, to test the Ecoindex you must have developed the application, so sobriety decisions must already be taken.

It will also be noted that the Ecoindex does not take into account certain performance indicators, but a site with poor performance will generate a feeling of obsolescence felt in the user, pushing him to unnecessarily renew his equipment.

Finally, the client-listed impact indicator does not consider JavaScript, which perhaps a source of more calculations than the DOM in certain situations.

Functional sobriety

After all, the functionality that the least impacts the environment is the one that is not developed. This saves time and money during development, and necessarily does not generate use

However, the decision to develop or not a functionality should not be taken lightly. The ideal for making this decision is to bring together a large part of the players on the site (developers, customers, users) and as far as possible to rely on reliable data (use data for example, which could have collected on a previous version of the platform).

Performance indicators

But what is the relationship between performance and eco -design? There are two points of interest.

The first is quite obvious, a more efficient application will probably consume fewer resources, or for less long. A way to test this assertion is to measure the consumption of the browser when using the web page with a tool like Powerapi.

The second has more impacts, but is less obvious. It is the feeling of obsolescence felt during the use of the site. A long loading time, the delay between the user's inputs and the machine's response, etc., are phenomena that can push the user to say "well, I would have to renew my equipment". Although it can be attractive to use the latest available features of the latest graphics cards or the latest processors on the market, it should not be forgotten that their use risks arouse frustration in a major part of users who have not These features

The rebound effect

The first example of a rebound effect is that of the James Watt steam engine which was much more economical in coal than the old models with equal performance. Rather than a drop in coal consumption, the gain in efficiency and the greater use of the steam engine to cause a constant increase in the use of coal.

This example is present from everywhere: a purchase of an LED bulb which consumes less leads to less vigilance on its use, 4G allows faster download (therefore less expensive for infrastructure) so we download more, etc.

We must therefore remain vigilant for a progress not followed by a change of habit which counterbalances the savings it allows.
