Was ist der EcoIndex?

EcoNIndex.fr ist ein Online -Service, der von Frédéric Bordage (Greenit.fr) entworfen und angeboten wird und von Gwenael Chaineu und Thomas Samain vom Unternehmen NXTWeb inszeniert wird.

Dieser Service zielt darauf ab, die Umweltauswirkungen einer Website zu identifizieren. Es bietet dem Benutzer mehrere Indikatoren:

  • Der berühmte EcoIndex: Umweltleistung (100 Punktzahl, höher ist besser)
  • Eine Notiz von A bis G widerspiegelt diese Leistung
  • Sein technischer Abdruck (Gewicht, Komplexität)
  • Sein ökologischer Fußabdruck (Treibhausgas, Wasser)
  • Eine Liste guter Praxis sowie eine Bewertung des Respekts dafür

Das Ziel des Dienstes ist ein Bewusstsein für die Umweltauswirkungen digitaler Dienste sowie der konkrete Nachverfolgungsvorschlag.

Technischer Abdruck

Der technische Fußabdruck einer Website wird durch mehrere Indikatoren dargestellt:

  • Die Größe des DOM: Dieser Indikator ermöglicht es, die Komplexität einer Website zu bewerten. Je größer das DOM, desto mehr benötigt es Strom und Speicher für seine Last sowie für alle Vorgänge in Bezug auf sie.
  • Das geladene Dateigewicht: Dieser Indikator bewertet die Auswirkungen des Ladens der Website im Netzwerk. Je schwerer die Dateien sind, desto mehr benötigen sie Netzwerkressourcen.
  • Die Anzahl der Anfragen zum Laden der Seite: Mit diesem Indikator können Sie beurteilen, wie viel das Laden der Seite Server erfordert. Diese Zahl ermöglicht den Verbrauch von Servern interpoliert.

Umweltabdruck

Der ökologische Fußabdruck wird in zwei Punkte unterteilt

  • Treibhausgasemission, gemessen in gleichwertigem CO2 -Gramm
  • Wasserverbrauch, gemessen in CL

Diese Werte stammen aus einer Schätzung unter Berücksichtigung aller Akteure (Kunden, Netzwerk und Server) und basieren auf einer Analyse des Lebenszyklus der verschiedenen Geräte.

Gute Praktiken

Das CNUMR bietet eine gute Übungsliste für das Design von Webplattform. Bestimmte Punkte in dieser Liste können während der Plattformanalyse automatisch getestet werden.

Eine nicht exexhustive Liste der getesteten guten Praktiken:

  • Die Anzahl des verwendeten JavaScript -Plugins.
  • Die Anzahl der CSS -Dateien. Es kann nützlich sein, Ihr CSS in mehrere spezialisierte Dateien zu schneiden. Dieser Schnitt kann gemäß einer funktionalen Logik (allgemeiner Inhalt, Seite A, Seite B usw.) oder einer Abfrage -Medienlogik (z. Der Browser lädt die erforderliche Seite sofort, und das ist mit einer sehr geringen Priorität nicht erforderlich. Seien Sie jedoch vorsichtig, dass die Ausschnitte relevant sind (weniger als ein Dutzend Dateien).
  • Die Anzahl des leeren SRC -Attributs. Bestimmter Browser (Chrome und Safari) verwaltet dieses Szenario schlecht und stellen eine zusätzliche Anfrage an die URL auf der Seite, um dieses Bild zu laden. Wie hier erläutert, besteht eine gute Möglichkeit, eine leere Quelle zu hinterlassen, das Attribut an Src = "Daten:".
  • Und ungefähr zwanzig andere Indikatoren.

Das ecoindex ein Ende darin?

Ist eine gute umweltverantwortliche Seite? Der EcoIndex ist ein guter Indikator, aber es ist oft notwendig, in einem Ökodesign -Prozess weiter zu gehen. Mit dem EcoIndex können Sie beispielsweise nicht die funktionale Nüchternheit einer Website testen. Um den EcoIndex zu testen, müssen Sie die Anwendung entwickelt haben, sodass bereits Nüchternheitsentscheidungen getroffen werden müssen.

Es wird auch angemerkt, dass der EcoIndex bestimmte Leistungsindikatoren nicht berücksichtigt, aber eine Site mit schlechter Leistung erzeugt ein Gefühl der Veralterung, das im Benutzer spürt ist und ihn dazu drängt, seine Ausrüstung unnötig zu erneuern.

Schließlich berücksichtigt der von Client gelistete Impact-Indikator JavaScript nicht, was in bestimmten Situationen möglicherweise eine Quelle für mehr Berechnungen als die DOM ist.

Funktionelle Nüchternheit

Schließlich ist die Funktionalität, die die geringste Auswirkungen auf die Umgebung beeinflussen, die nicht entwickelt. Dies spart Zeit und Geld während der Entwicklung und erzeugt notwendigerweise keine Verwendung

Die Entscheidung, eine Funktionalität zu entwickeln oder nicht, sollte jedoch nicht leicht genommen werden. Das Ideal für diese Entscheidung besteht darin, einen großen Teil der Spieler der Website (Entwickler, Kunden, Benutzer) zusammenzubringen und sich so weit wie möglich auf zuverlässige Daten zu verlassen (zum Beispiel Daten verwenden, die in einer früheren Version hätte erfassen können der Plattform).

Leistungskennzahl

Aber wie ist die Beziehung zwischen Leistung und Ökodesign? Es gibt zwei Punkte von Interesse.

Die erste ist ziemlich offensichtlich, eine effizientere Anwendung wird wahrscheinlich weniger Ressourcen oder weniger lange konsumieren. Eine Möglichkeit, diese Behauptung zu testen, besteht darin, den Verbrauch des Browsers bei der Verwendung der Webseite mit einem Tool wie PowerAPI zu messen.

Die zweite hat mehr Auswirkungen, ist aber weniger offensichtlich. Es ist das Gefühl der Veralterung während der Nutzung der Website. Eine lange Ladezeit, die Verzögerung zwischen den Eingängen des Benutzers und der Reaktion der Maschine usw. sind Phänomene, die den Benutzer dazu bringen können, zu sagen: "Nun, ich müsste meine Ausrüstung erneuern." Obwohl es attraktiv sein kann, die neuesten verfügbaren Funktionen der neuesten Grafikkarten oder der neuesten Prozessoren auf dem Markt zu verwenden, sollte nicht vergessen werden, dass ihre Nutzungsrisiken in einem Großteil von Benutzern, die diese Funktionen nicht haben, frustriert werden

Bewertung der Auswirkungen von JavaScript

Was ist JavaScript auf einer Webseite? Validieren Sie ein Formular, aktualisieren Sie die Seite, führen Sie Ajax -Anfragen aus, untergraben die Kryptowährung. Im JavaScript sind immer mehr möglich, aber ist es im Kontext aller Projekte nützlich?

Selbst im Dienstprogramm können wir mehrere Fragen stellen, z. B. die Verteilung der Berechnungen zwischen dem Kunden und dem Server.

Der Rückpralleffekt

Das erste Beispiel für einen Rebound -Effekt ist die James Watt -Dampfmaschine, die in Kohle viel wirtschaftlicher war als die alten Modelle mit gleicher Leistung. Anstelle eines Abfalls des Kohleverbrauchs, des Effizienzgewinns und der stärkeren Nutzung der Dampfmaschine, um eine konstante Zunahme der Verwendung von Kohle zu erhöhen.

Dieses Beispiel ist von überall vorhanden: Ein Kauf einer LED -Glühbirne, die weniger zu weniger Wachsamkeit bei seiner Verwendung führt, ermöglicht 4G einen schnelleren Download (daher kostengünstiger für die Infrastruktur), also laden wir mehr herunter usw.

Wir müssen daher wachsam bleiben für einen Fortschritt, der nicht von einem Gewohnheitswechsel gefolgt ist, der die Ersparnisse ausgleichet.
